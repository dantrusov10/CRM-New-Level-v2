# Контракт: даты в контексте для любого вызова ИИ из CRM

## Проблема
ИИ видит текст комментариев без явной привязки к «сегодня» и без сортировки по времени события. В результате устаревшие фразы («клиент в отпуске до конца января») могут трактоваться как актуальные план на будущее.

## Обязательное правило для всех эндпоинтов ИИ
Каждый запрос к LLM (помощь, исследование, анализ сделки, апдейт, дашборд и т.д.) **должен** включать два блока:

### 1) Момент запроса (reference «сейчас»)
- `request_instant_utc` — ISO-8601 в UTC на момент **отправки запроса в провайдера** (не время создания сделки).
- Опционально: `request_timezone` / локальная дата для отображения менеджеру (но для модели достаточно одного UTC ISO).

### 2) Хронология событий с датами
- Для сделки: выборка из `timeline` (и при необходимости задач/этапов), где у **каждой** строки явно указана дата события.
- Поле для даты события (в порядке приоритета): `timestamp` → `created`.
- Формат строки в тексте для модели: `[YYYY-MM-DD] action: comment` (дата из `timestamp`/`created`, нормализованная в UTC или явно помеченная).
- Сортировка: **от новых к старым** (сначала самое свежее).
- Лимит: разумный cap (например 50–120 строк) + обрезка длины `comment`, чтобы не съесть весь контекст.

## Инструкция модели (system / первый user-блок)
После фактов вставить короткую инструкцию (на русском), например:

- «`request_instant_utc` — это текущий момент запроса к ИИ.»
- «Все даты в блоке хронологии — даты **событий в CRM**, не «сегодня».»
- «Если в старом комментарии описано будущее относительно даты этого комментария, сравни его с `request_instant_utc` и с более новыми событиями; не предлагай ждать то, что по времени уже должно было наступить, если нет более свежего подтверждения обратного.»

## Где внедрять
- **Control plane / AI gateway** (`platform-console`, путь на сервере см. `AGENT_CONTEXT.md`): центральная точка — все вызовы LLM проходят через один helper, который **всегда** добавляет блок дат.
- **CRM frontend**: при появлении прямых вызовов к LLM из браузера — использовать `apps/web/src/lib/aiDateContext.ts` (тот же смысл, что и в Python-референсе).

## Реализации в репозитории
- TypeScript: `apps/web/src/lib/aiDateContext.ts`
- Python (референс для gateway): `tools/gateway_ai_date_context.py`
