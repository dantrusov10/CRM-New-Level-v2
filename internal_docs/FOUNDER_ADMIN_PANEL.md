# FOUNDER / ADMIN PANEL (Control Plane) — AS IS и целевая рамка

## Зачем нужен ЛК фаундера
Единая точка управления клиентскими средами CRM: доступы, лимиты AI, политика моделей, биллинг, здоровье инстансов.

## Что есть в текущем репозитории
- Админ-раздел CRM-instance:
  - пользователи/роли,
  - воронка,
  - конструктор полей,
  - parser-настройки,
  - KPI/КП каркас.
- Отдельный founder-level control plane как полноценный runtime-контур в этом репо отсутствует.

## Какие разделы должны быть в founder/control plane
1. Клиенты (tenants): список, статус, тариф, домены, owner.
2. Инстансы: shared/dedicated, health, версии схемы.
3. AI: модели, маршрутизация, лимиты, usage, ошибки.
4. Биллинг: план, квоты, consumption, платежный статус.
5. Безопасность: политики данных, аудит, ключи интеграций.
6. Операции: provisioning, backup, restore, инциденты.
7. Парсеры: источники, токены, расписания, SLA.

## Управление клиентами
- Создание, блокировка/разблокировка, архивирование.
- Назначение админов клиента.
- Привязка к тарифу и инфраструктурному профилю.

## Управление тарифами
- Ограничения пользователей/сделок/хранилища.
- AI-лимиты: запросы, токены, бюджеты.
- Доступные функции по тарифу (feature flags).

## Управление лимитами AI
- Лимит на tenant / период.
- Лимит на пользователя.
- Hard-stop и soft-warning.
- Автологика при перерасходе.

## Управление инстансами
- Shared: несколько клиентов в едином кластере.
- Dedicated: отдельный VPS/PocketBase для клиента.
- Контроль версии схемы и совместимости frontend/backend.

## Создание новой клиентской среды
- Автопровиженинг коллекций.
- Инициализация ролей и базовых настроек.
- Создание первичного admin-пользователя.

## Блокировка/разблокировка клиента
- Флаг состояния в control plane.
- Режим только чтения / полный блок.
- Аудит причины и инициатора.

## Просмотр использования AI
- По tenant/user/model/day.
- Токены, стоимость, ошибки, latency.
- Топ сценариев расхода.

## Просмотр ошибок
- Ошибки провайдеров, таймауты, валидация output.
- Ошибки парсеров и импортов.
- Ошибки доступов/правил.

## Управление моделями
- Белый список моделей.
- Политики per-тариф/per-tenant.
- Failover/priority маршрутизация.

## Управление системными промтами
- Versioned prompt templates.
- Scope: global / tenant / scenario.
- Rollback и история изменений.

## Управление парсерами
- Источники, платформы, расписания.
- API keys (`ENV_SECRET_REQUIRED`).
- Контроль частоты и качества результатов.

## Управление платежами
- План, период, выставление счетов, статусы.
- Блокировки по неуплате.

## Управление доменами
- Привязка кастомного домена клиента.
- SSL и маршрутизация.

## Мониторинг здоровья CRM-инстансов
- Доступность API/DB.
- Ошибки 5xx/latency.
- Свежесть данных и jobs.
